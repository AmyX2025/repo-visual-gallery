memory-lancedb-pro 到底是什么东西？

简单说，它是给 OpenClaw（你的 AI 小龙虾）装的一个「超级记忆大脑」。 普通的记忆系统就像用便签纸记事，而这个插件相当于给你的 AI 装了一个带搜索引擎的图书馆 —— 不仅能记住东西，还能用多种方式精准地找到以前记过的内容。

memory-lancedb-pro/
├── index.ts                <- 插件的入口，相当于图书馆的大门
├── cli.ts                  <- 管理员工具，命令行管理记忆
├── openclaw.plugin.json    <- 给 OpenClaw 的安装说明书
├── src/
│   ├── store.ts            <- 仓库管理员，负责存取记忆
│   ├── retriever.ts        <- 搜索引擎，混合检索的核心
│   ├── embedder.ts         <- 翻译官，把文字变成数字向量
│   ├── scopes.ts           <- 门禁系统，控制谁能看什么
│   ├── noise-filter.ts     <- 垃圾分拣员，过滤无用信息
│   ├── adaptive-retrieval.ts<- 智能开关，判断是否需要搜记忆
│   ├── access-tracker.ts   <- 图书借阅记录员
│   ├── chunker.ts          <- 切割工，把长文档切成小块
│   ├── tools.ts            <- AI 的记忆工具箱
│   └── migrate.ts          <- 搬家工，从旧系统迁移数据
├── skills/lesson/SKILL.md   <- 经验提取技能
└── examples/                <- 示例：聊天记录自动提炼记忆

架构层级

点击每一层查看详细说明。从上到下，就像一栋楼的楼层 —— 用户在顶层，底层是基础设施。

1
🦞 OpenClaw 网关 你的 AI 助手本体
OpenClaw 就是你的 AI 小龙虾本体。它通过 WhatsApp、Telegram 等渠道跟你聊天。当它需要「想起」以前的事情时，就会调用这个记忆插件。可以把 OpenClaw 想象成一个很聪明但记性不好的朋友 —— 这个插件就是给它配的外置硬盘。
2
🔌 插件入口 注册 + 生命周期管理
index.tsopenclaw.plugin.json
index.ts 是整个插件的大门。它负责向 OpenClaw 注册自己（就像新员工入职），解析配置文件，然后在关键时刻插入钩子：AI 开始回答前自动搜记忆、AI 回答完后自动存记忆、用户开新会话时提炼旧对话中的经验。
3
🧰 AI 工具层 记忆的存、取、删、改
src/tools.tssrc/scopes.ts
这一层提供了 AI 可以直接使用的四个核心工具：记住（memory_store）、回忆（memory_recall）、遗忘（memory_forget）、更新（memory_update）。同时还有门禁系统（scopes），确保不同的 AI 代理只能访问自己被授权的记忆，就像公司里不同部门有不同的文件柜权限。
4
🔍 混合检索引擎 多策略搜索 + 评分排序
src/retriever.tssrc/noise-filter.tssrc/adaptive-retrieval.tssrc/access-tracker.ts
这是整个插件最核心的部分。它同时用两种方式搜索记忆：语义搜索（理解「意思」）+ 关键词搜索（精确匹配字词），然后把结果合并。接着经过 6 道评分流水线 —— 包括新鲜度加成、重要性加权、长度惩罚、时间衰减、去重过滤 —— 最终挑出最相关的几条记忆。就像图书馆不只按书名找书，还综合考虑借阅频率、出版年份、与你问题的相关度。
5
💾 存储层 LanceDB 向量数据库
src/store.tssrc/chunker.tssrc/migrate.ts
存储层用 LanceDB 这个向量数据库来保管所有记忆。每条记忆被存储为一行数据，包含原文、向量（数字化的语义指纹）、分类、权限范围、重要性等信息。LanceDB 是嵌入式数据库，直接存在你服务器的硬盘上，不需要额外启动数据库服务。chunker 负责把太长的内容切成小块再存。
6
🌐 外部服务层 嵌入 API + 重排序 API
src/embedder.ts
要让记忆可以被「按意思搜索」，首先需要把文字翻译成数字向量 —— 这个翻译工作交给外部的嵌入服务（Jina、OpenAI 等）。同时，搜索结果还可以交给重排序服务做二次精选，就像先用搜索引擎粗选，再请专家帮你精挑。这些都是通过网络调用的云端 AI 服务。

核心工作流：记忆检索全过程

当你问 AI 一个问题，插件会在后台走完这些步骤，把最相关的记忆找出来。

1
智能判断
先看看你的问题是否需要翻记忆。「你好」这类打招呼就跳过，复杂问题才启动搜索。
→
2
语义翻译
把你的问题通过嵌入 API 翻译成一串数字向量，方便后续按「意思」搜索。
→
3
双路搜索
同时启动两路搜索：向量搜索（按意思找）+ BM25 搜索（按关键词找），然后合并结果。
→
4
精选重排
调用重排序 AI 对候选结果做二次评分，再经过新鲜度、重要性等 6 道评分流水线。
→
5
注入上下文
把最终选出的记忆（最多 3 条）注入 AI 的上下文，AI 就能「记起」相关信息来回答你。

外部工具箱

这个插件用到的第三方服务和工具，以及它们的费用。

📊 LanceDB 免费
AI 的「记忆仓库」。一个嵌入式的向量数据库，直接存在你服务器硬盘上，不用额外启动数据库服务。它能同时支持按「意思」搜索和按「关键词」搜索。
角色：记忆的存储和检索引擎
免费额度：完全免费，开源无限制
价格：开源版 $0；云托管版从 $16/月起
何时升级：只有需要云端托管和团队协作时才考虑付费版
关联：存储层、检索引擎
🧠 Jina AI 嵌入 免费起步
把文字翻译成「数字指纹」的翻译官。它能理解文字的含义，把意思相近的内容翻译成相近的数字，这样就能按「意思」搜索了。
角色：文本嵌入（向量化）服务
免费额度：每月 10 万个字符单位
价格：约每百万字 $0.02
何时升级：每天存取记忆超过几百次时
关联：翻译官（embedder.ts）
🤖 OpenAI 嵌入 按量付费
OpenAI 出品的另一个「文字翻译官」。和 Jina 做的事情一样，但由不同公司提供。可以二选一，也可以作为备用。
角色：备选嵌入服务
免费额度：新用户 $5 额度（约能处理 250 万条短文本）
价格：约每百万字 $0.02
何时升级：$5 额度用完后自动按量计费
关联：翻译官（embedder.ts）
⚖️ Jina 重排序 免费起步
搜索结果的「裁判」。初步搜索出一批候选记忆后，它会更仔细地比较每条记忆和你问题的相关度，重新排序，把最相关的排到最前面。
角色：交叉编码器重排序
免费额度：与嵌入共享免费额度
价格：约每百万字 $0.02
何时升级：可选功能，关闭后使用免费的本地余弦相似度替代
关联：检索引擎（retriever.ts）

角色：运行时环境

术语词典

页面中出现的技术术语，都用大白话解释一遍。

向量嵌入 (Vector Embedding)
把一段文字翻译成一串数字（比如 1024 个小数）。意思相近的两句话，翻译出来的数字也会很接近。这样计算机就能通过比较数字来判断「这两段话说的是不是一回事」。
LanceDB
一个「随身携带的智能仓库」。跟传统数据库不同，它不需要单独启动一个服务器程序，直接嵌入到你的应用里。它特别擅长存储和搜索那些翻译成数字的文本。
BM25
一种经典的关键词搜索算法，像搜索引擎那样按「字词匹配」找内容。比如你搜「虾浪漫」，它会找所有包含这几个字的记忆，出现次数多的排前面。
混合检索 (Hybrid Retrieval)
同时使用「按意思搜」和「按关键词搜」两种方式，然后把结果合在一起。就像找餐厅时既看评分（语义）又看菜名（关键词），比只用一种方式更准确。
交叉编码器重排序 (Cross-Encoder Rerank)
初步搜出一堆候选结果后，再用一个更聪明的 AI 模型仔细比对每条结果和你的问题。就像面试先筛简历（初步搜索），再面谈（重排序），最后录用最合适的。
作用域 (Scope)
记忆的「权限标签」。比如标记为 global 的记忆所有人都能看，标记为 agent:main 的只有主代理能看。就像公司里的文件分「公开」「部门内」「仅限个人」。
互惠秩融合 (RRF)
一种合并多个搜索结果排名的数学方法。如果一条记忆在「按意思搜」和「按关键词搜」中都排名靠前，融合后它的综合排名会更高。
最大边际相关性 (MMR)
防止搜索结果都说同一件事的去重机制。如果前 3 条记忆都在说「你喜欢用 Claude」，它会保留 1 条，把另外 2 条替换成其他有用信息。
时间衰减 (Time Decay)
越老的记忆分数越低，就像人的记忆会随时间变淡。默认 60 天减半。但如果一条记忆经常被翻出来用，它的「保质期」会延长 —— 就像常用的知识不容易忘。
噪声过滤 (Noise Filter)
自动识别并丢弃没有记忆价值的内容。比如 AI 说的「我没有个人偏好」、用户的「好的」「知道了」这类内容，存下来只是浪费空间。
嵌入 API
云端的「翻译服务」。你把一段文字发过去，它返回一串数字。就像找翻译公司翻译文件，你付费按字数收费。Jina、OpenAI 都提供这种服务。
生命周期钩子 (Lifecycle Hook)
在特定时刻自动执行的操作。比如 AI 回答之前（自动搜记忆）、AI 回答之后（自动存记忆），就像公司规定「每天下班前要写日报」。
插件 (Plugin)
给软件加新功能的「外挂模块」。就像手机装 App 一样，OpenClaw 通过安装不同插件来获得新能力。memory-lancedb-pro 就是一个记忆增强插件。
systemd
Linux 系统的「总管家」。它负责管理所有后台运行的程序，确保它们开机自动启动、崩溃了自动重启。OpenClaw 通常靠它来保持 24 小时运行。
