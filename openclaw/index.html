GitHub 开源项目解读

OpenClaw 到底是什么东西？

想象你有一个永远在线的私人助手，它同时帮你回 WhatsApp、Telegram、Discord、Slack、Email......所有消息。 OpenClaw 就是这样一个开源 AI 助手框架 -- 它跑在你自己的设备上，数据完全归你，还能通过 5700+ 个"技能"学会各种新本事。 在 GitHub 上拿了 25.8 万颗星，比大部分明星项目都火。

PROJECT STRUCTURE

Architecture

架构全景：从上到下六层积木

点击每一层查看详细解读。整个系统像一栋六层大楼，从用户触达到 AI 模型调用，每层各司其职。

1
📱
用户触达层 -- 消息从哪里来
20+ 个聊天平台
WhatsAppTelegramDiscord SlackSignaliMessage TeamsMatrixEmail Google ChatLINEFeishu
这一层就像一个超级前台 -- 不管客人从哪扇门进来（WhatsApp、Telegram、还是 Email），前台都能听懂他们说的话，然后翻译成统一的格式往里传。每个平台都有一个专属的"翻译官"（Channel Adapter），它负责把平台特有的消息格式转换成 OpenClaw 内部通用的格式。核心代码在 src/channels/ 和 extensions/ 里，支持的平台超过 20 个。
2
⚡
Gateway 网关层 -- 总调度塔台
实时通信控制中枢
server.impl.tsserver-http.ts server-channels.tsauth.ts protocol/node-registry.ts
Gateway 是整个系统的心脏，像机场的塔台一样指挥一切。所有消息进来先经过它，所有 App 连接也走它。它使用 WebSocket 技术（一种让设备和服务器保持"电话连线"的方式）保持实时通信。它负责身份验证（谁可以连？）、消息路由（这条消息该给哪个 AI？）、Session 管理（之前聊了什么？），以及设备配对（你的 iPhone 要先"握手"才能用）。代码在 src/gateway/，是整个项目最庞大的模块。
3
🧠
Agent 智能体层 -- AI 真正干活的地方
AI 思考 + 执行任务
pi-embedded-runner/pi-tools.ts bash-tools.tsskills/ system-prompt.tsmodel-auth.ts
这是 AI 大脑所在的楼层。Agent 基于 Pi SDK（AI 建造工具包）构建，能调用各种工具 -- 执行电脑命令、上网查资料、操控浏览器、生成网页界面，还能根据你装的 Skills 学会新技能。每次对话开始前，它会先写一份"身份说明"告诉 AI 大脑"你是谁、你能做什么"。AI 的记忆容量有限（用 Token 计量），记不住太多内容时会自动把旧对话压缩成摘要。任务太复杂？它还能派出"分身"分头处理。
4
🔌
服务支撑层 -- 记忆、浏览器、定时任务
记忆 + 浏览器 + 定时任务
memory/browser/canvas-host/ cron/tts/media/
这一层是 Agent 的"后勤部队"。记忆系统用 SQLite（一种轻量级的本地数据库，像随身笔记本）存储，支持两种查找方式 --"按关键词找"（比如搜索"天气"两个字）和"按意思找"（比如你问"温度"，它能联想到之前聊过的"天气预报"话题）。Playwright（AI 的"机器人手"）能替 AI 操控浏览器上网查资料、填表格。Canvas 让 AI 能画出可交互的网页界面。Cron 是"定时闹钟"，支持定时任务（比如每天早上 8 点给你发天气预报）。TTS（文字转语音）让 AI 能"开口说话"。
5
💻
客户端 App 层 -- 各平台原生体验
macOS + iOS + Android + Web UI
apps/macos/ (Swift)apps/ios/ (Swift) apps/android/ (Kotlin)ui/ (Lit)
除了通过第 1 层的 20+ 个聊天平台（WhatsApp、Telegram 等）使用之外，OpenClaw 还开发了自己的专属 App。macOS 版是一个住在屏幕右上角菜单栏里的小助手，随时点击就能对话；iOS 和 Android 都有独立的 OpenClaw App，装在手机上就能直接跟你家里的 Gateway 通信。另外还有一个网页版控制面板，打开浏览器就能监控和管理 AI 助手的运行状态。更酷的是，这些自家 App 还能让 AI "借用"你手机上的硬件 -- 比如用摄像头拍照、用麦克风录音，这是第三方聊天平台做不到的。
6
☁
外部服务层 -- AI 模型 & 第三方 API
10+ 模型供应商 + 搜索/语音
OpenAIAnthropicGemini OllamaBrave SearchElevenLabs DeepgramFly.io
最底层连接着各种外部服务。AI "大脑"方面支持 10+ 家供应商（OpenAI、Anthropic、Google Gemini 等等），还有自动故障切换功能 -- 就像手机有两张 SIM 卡，一张没信号了自动切换到另一张。每家服务都需要一把"数字钥匙"（API Key）来使用。搜索用 Brave Search，语音转文字用 Deepgram（AI 速记员），文字转语音用 ElevenLabs（AI 配音演员）。部署可以选 Fly.io 云端或者自己用 Docker 在本地跑。

Behind the Scenes

组件之间怎么"聊天"的？

如果 OpenClaw 的各个组件是同事，它们在工作群里是怎么协作的？

场景一：收到一条 WhatsApp 消息
场景二：Agent 需要上网查资料
场景三：手机拍照指令
WA
WhatsApp 翻译官
报告！刚收到一条消息，来自 +86 138****1234："明天北京天气怎么样？"。已经翻译成统一格式了，交给你处理。
WhatsApp 连接工具（Baileys）收到消息后，翻译官把 WhatsApp 的专用格式转换成 OpenClaw 内部通用格式
GW
Gateway 塔台
收到！让我先查一下......这个号码在允许列表里，验证通过。Session Key 是 main:whatsapp:+8613800001234，上次聊天记录已加载。Agent，轮到你了！
Gateway 检查这个号码是否在"允许名单"里，然后加载之前的聊天记录文件，把历史对话一起打包给 AI
AI
Pi Agent
好的，我先看看有没有天气相关的 Skill......没有。那我用 web_fetch 工具去查一下天气预报网站。稍等！
Agent 先检查自己有没有天气相关的技能，没有就决定直接上网去查
AI
Pi Agent
查到了！北京明天多云转晴，最高温 18 度，最低温 6 度。我把回复组织好了，Gateway 帮我发回去吧。
GW
Gateway 塔台
搞定！已经通过 WhatsApp 翻译官发回去了，用户那边应该已经看到回复了。Session 已保存。
回复经 Gateway 送回 WhatsApp 翻译官，再通过 WhatsApp 连接工具发出去，聊天记录自动保存到历史文件中
AI
Pi Agent
用户让我帮忙查一下 React 19 的新特性。这个我需要上网看看最新资料。浏览器同学，出动！
Agent 判断这个问题需要最新信息，决定让"机器人手"（Playwright）帮忙上网查
BR
Playwright 机器人手
好嘞！我在后台悄悄打开了一个浏览器（不用显示画面的那种），访问了 react.dev 官网。我不截图 -- 我直接读取页面的"文字骨架"（Accessibility Tree），信息一样全但省钱 100 倍。内容拿到了，给你！
Playwright（AI 的机器人手）启动了一个"隐形浏览器"（Chromium），通过读取网页的文字结构而非截图来获取内容，极大节省费用
AI
Pi Agent
太好了，信息很全。让我整理一下......React 19 主要有 Server Components 正式版、Actions API、use() Hook 等新特性。我把总结写好了，发给用户吧。
MEM
Memory 记忆模块
我把这次查到的 React 19 信息存进"记忆笔记本"了。下次用户再问相关问题，我能直接翻笔记本找到，不用再上网查。
Memory 模块把信息存入本地数据库（SQLite），下次类似的问题可以按"意思"联想检索，不需要关键词完全匹配
AI
Pi Agent
用户说"帮我拍张照片"。这个需要用到手机摄像头......Gateway，有哪些 Node 在线？
GW
Gateway 塔台
让我查一下 Node Registry......找到了！用户的 iPhone 15 Pro 在线，注册了 camera 能力。我把拍照请求路由过去。
Gateway 在"设备花名册"里查找哪些已配对的设备有摄像头能力
IP
iPhone Node
收到拍照指令！正在打开后置摄像头......咔嚓！照片拍好了，JPEG 数据通过 WebSocket 回传给 Gateway。
iPhone 上的 App 收到拍照指令后，调用手机摄像头拍照，然后把照片通过实时连接传回 Gateway
AI
Pi Agent
照片收到了！已经发给用户了。如果用户需要，我还能分析照片内容。

Glossary

术语速查词典

页面上出现的所有技术术语，用最简单的话解释给你听。

Gateway
整个系统的"总调度塔台"。所有消息进出、App 连接、Agent 调度都经过它。就像机场的控制塔，所有飞机的起降都归它管。默认跑在本地 18789 端口。
WebSocket
一种实时双向通信协议。普通 HTTP 像发短信（你问一句我答一句），WebSocket 像打电话（双方随时都能说话）。Gateway 用它来保持跟所有客户端的实时连接。
Agent
AI 智能体，就是那个真正"思考"和"干活"的角色。它能读懂你的消息、决定要不要使用工具、然后生成回复。可以理解为一个有自主判断力的 AI 员工。
Channel Adapter
消息平台的"翻译官"。每个聊天平台（WhatsApp、Telegram......）说的"方言"不一样，翻译官把它们统一翻译成 OpenClaw 能懂的"普通话"。
Session
一次对话的上下文记录。就像你跟朋友的聊天记录 -- 它记住了你们之前说了什么，这样 AI 才能理解"上次提到的那个"指的是什么。存在 JSONL 文件里。
Skill
Agent 的"技能配方"。就像游戏角色学的技能 -- 每个 Skill 教会 Agent 做一件特定的事（比如操作 GitHub、查 1Password 密码）。可以从 ClawHub 市场下载。
Pi SDK
OpenClaw 用来建造 AI 助手的"建筑工具包"（SDK 就是"软件开发工具包"的缩写）。它把对话管理、工具调用、AI 大脑对接这些复杂的事情打包好了，开发者只需要拼积木就行。就像盖房子不用自己烧砖，直接用预制构件搭建。
Node
连接到 Gateway 的设备（你的手机、电脑等）。不只是被动接收消息，还能主动"贡献"自己的能力 -- 比如手机可以贡献摄像头拍照、麦克风录音。AI 助手需要时就能"借用"你手机上的这些硬件。
Pairing
设备配对，类似蓝牙耳机第一次连手机时要"配对"。新设备想连你的 Gateway，需要你批准一个配对码，防止陌生人偷偷连上来。
Context Window
AI 模型一次能"看到"的信息量上限，用 Token 计算。就像人的短期记忆容量 -- 塞太多东西就记不住了。OpenClaw 会自动压缩旧对话来节省空间。
Failover
故障自动切换。一个 API Key 用完了或者挂了？没关系，自动切换到下一个。就像你有好几张电话卡，一张没信号了就换一张。
Accessibility Tree
网页的"文字版简历"。一个网页有图片、颜色、动画等花花绿绿的元素，但 Accessibility Tree 把这些都去掉，只保留纯文字结构 -- 比如"这里有个按钮叫'登录'"、"这里有个标题叫'新闻'"。AI 读这个比看截图快 100 倍，还省钱（省 Token）。
JSONL
一种简单的数据存储格式。JSON 是计算机界的"通用数据语言"（就像 Excel 是给人看的表格，JSON 是给程序看的表格），而 JSONL 就是"每行一条 JSON"。OpenClaw 用它来存对话历史 -- 像日记本一样，每次对话往后面追加一行。
Docker
把软件打包进"标准集装箱"的工具。不管你的电脑是 Mac、Linux 还是 Windows，集装箱里的东西到哪都能原样运行。一行命令启动整个 OpenClaw。
ClawHub
OpenClaw 的"技能市场"，有 5,700+ 个社区制作的 Skill 可以下载安装。就像手机的 App Store，但装的是 AI 能力而不是 App。
Tailscale
给设备建"私密隧道"的工具。你在咖啡店也能安全连回家里的 Gateway，不用开端口、不用折腾路由器。就像给你的手机和家里电脑之间挖了一条只有你能走的地下通道。
Playwright
AI 的"机器人手"，能替 AI 操控浏览器。就像你自己用鼠标打开网页、点按钮、填表格一样，Playwright 能代替 AI 做这些事 -- 只不过速度快得多，而且不需要真的有一块屏幕显示出来。
API Key
一把"数字钥匙"。当 OpenClaw 想要使用 OpenAI、Anthropic 这些外部服务时，需要出示这把钥匙来证明身份并计费。就像进健身房要刷会员卡 -- 没有卡就进不了门，刷了卡才知道该扣谁的钱。
Token
AI 读取和生成文字的计量单位，大约 1 个中文字 = 1-2 个 Token。AI 服务商按 Token 数量收费，就像手机流量按 MB 计费一样。AI 的"记忆力"（上下文窗口）也用 Token 衡量 -- 记得住多少字，取决于 Token 上限。
Cron
"定时闹钟"系统。你可以设置让 AI 助手在固定时间自动做某件事，比如每天早上 8 点发天气预报、每周五下午提醒你写周报。就像手机上的重复闹钟，到点自动响。
TTS (Text-to-Speech)
"文字转语音"技术。就是让 AI 能"开口说话" -- 把文字内容变成真人般的语音朗读出来。就像导航软件里的语音播报，但声音更自然、更像真人。
Chromium
Google Chrome 浏览器的"开源版本"。你平时用的 Chrome 就是基于 Chromium 开发的。OpenClaw 在后台用它来帮 AI 浏览网页，只不过是"无头模式"运行 -- 也就是不显示窗口画面，只在后台默默干活。
Canvas
AI 的"画板"。普通 AI 只能回复文字，但通过 Canvas，OpenClaw 的 AI 助手能画出真正可以点击、可以交互的网页界面 -- 比如给你画一个待办清单、一个图表、或者一个小工具。就像 AI 有了一块白板，能边聊天边画给你看。
Baileys
一个让程序能收发 WhatsApp 消息的开源工具。WhatsApp 官方不提供这种接口给个人开发者，Baileys 通过"破解"WhatsApp 网页版的通信方式实现了同样的效果 -- 相当于让你的 AI 助手也有了一个 WhatsApp 账号。
System Prompt
给 AI 的"岗位说明书"。在你跟 AI 对话之前，系统会先悄悄告诉 AI 一段话，内容大致是"你是谁、你能做什么、你不能做什么"。就像新员工入职第一天收到的岗位职责说明，它决定了 AI 的性格和行为边界。
