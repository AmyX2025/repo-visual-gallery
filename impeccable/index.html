Impeccable 到底是什么东西？

简单来说，它是一套"设计词汇表"，专门教 AI 编码助手（比如 Claude、Cursor、Gemini）怎么做出好看的界面。因为 AI 生成的网页往往千篇一律 -- 一样的字体、一样的紫色渐变、一样的卡片套卡片。Impeccable 用 1 个核心设计技能、17 条命令和一堆"千万别这样做"的反面教材，让 AI 学会真正的设计品味。
impeccable/
  source/                     <- 所有设计知识的"大脑"，唯一的真相来源
    skills/
      frontend-design/       <- 核心设计技能，附带 7 本"参考手册"
        SKILL.md               <- 设计主技能文件（该做什么、别做什么）
        reference/             <- 7 个专业领域的深度知识
          typography.md        <- 字体排版指南
          color-and-contrast.md<- 配色和对比度指南
          spatial-design.md    <- 空间布局指南
          motion-design.md     <- 动效设计指南
          ...
      audit/ polish/ critique/ <- 17 个命令各一个文件夹
  scripts/                    <- 构建系统，像工厂里的流水线
    build.js                   <- 总调度员，一键把源码变成各平台能用的格式
    lib/transformers/        <- 6 个"翻译官"，负责适配不同 AI 工具
      cursor.js                <- Cursor 翻译官
      claude-code.js           <- Claude Code 翻译官
      gemini.js                <- Gemini CLI 翻译官
      codex.js                 <- Codex CLI 翻译官
      agents.js                <- VS Code Copilot 翻译官
      kiro.js                  <- Kiro 翻译官
  dist/                       <- 成品仓库，用户直接下载这里的文件
  server/                     <- 本地开发服务器
  public/                     <- 官网 impeccable.style 的源码
  functions/                  <- 线上 API 接口（部署在云端）
  package.json                 <- 项目身份证
  CLAUDE.md                    <- 给 AI 助手看的项目说明
Architecture

项目架构全景图

Impeccable 的核心逻辑：在一个地方写好所有设计知识，然后用构建系统自动"翻译"成 6 种 AI 工具各自能理解的格式。点击每一层查看详情。

1
🎨
设计知识层 (Source)
所有设计智慧的源头，唯一的真相来源
source/skills/SKILL.mdreference/*.md
▼
这一层就像一个"设计大师的笔记本"。里面装着 18 个技能定义（1 个核心设计技能 + 17 个命令技能），外加 7 本深度参考手册，涵盖字体排版、配色理论、空间布局、动效设计、交互模式、响应式适配和文案写作。所有内容都用带有元数据头部的 Markdown 格式编写，是整个项目的"单一信息源"。最关键的是它还包含一组"反面教材"列表 -- 明确告诉 AI 哪些设计模式已经被用烂了，必须避免。
2
🔧
构建系统层 (Build System)
把设计知识"翻译"成各平台格式的工厂流水线
scripts/build.jslib/utils.jslib/zip.js
▼
构建系统是整个项目的"翻译工厂"。build.js 是总调度员，它先读取所有源文件，然后调用 6 个专门的翻译器把内容转换成不同 AI 工具能理解的格式。工具层 utils.js 负责解析元数据头部、处理占位符替换（比如把 {{model}} 替换为 "Claude" 或 "Gemini"），zip.js 负责把成品打包成 ZIP 下载包。整个系统用 Bun 运行，速度飞快。构建还会同时编译官网的样式表和静态页面。
3
🔄
转换器层 (Transformers)
6 个"翻译官"，每人负责一种 AI 工具的方言
cursor.jsclaude-code.jsgemini.jscodex.jsagents.jskiro.js
▼
每个翻译器大约 30-85 行代码，职责非常专一：读取源文件，应用该平台的格式规则，然后输出。比如 Claude Code 支持完整的元数据头部和参数，就原样保留；Cursor 不支持元数据头部，就把那部分去掉；Gemini 使用 TOML 格式的命令定义，就做相应转换；Codex 的参数语法不同（把 {{target}} 变成 $TARGET），也要单独处理。每个翻译器还支持"前缀模式"（在所有命令前加 i-），方便用户避免和已有命令冲突。
4
📦
分发层 (Distribution)
成品仓库，用户下载安装的就是这里的文件
dist/cursor/dist/claude-code/dist/gemini/dist/codex/dist/universal/
▼
分发层是用户真正接触到的部分。构建完成后，每个 AI 工具都有一个专属目录，里面放着该工具能直接识别的技能文件。还有一个"通用包"，把所有平台的文件合在一起。用户只需要下载对应的 ZIP 包，解压到项目根目录就能用。整个 dist/ 目录会被提交到代码仓库，这样用户连构建工具都不需要安装。
5
🌐
官网与 API 层 (Website & API)
impeccable.style 官网 + 下载接口
public/server/functions/
▼
官网用纯粹的原生网页技术（没有任何前端框架）构建，追求极致的简洁。样式用 Tailwind CSS 编译，配色使用现代的 OKLCH 色彩空间，字体选了 Cormorant Garamond 和 Instrument Sans 的组合。开发时用 Bun 自带的服务器，上线后部署在 Cloudflare Pages。API 接口提供技能列表查询、单文件下载和整包下载功能，开发环境和线上环境共享同一套处理逻辑，零重复代码。
6
☁
外部服务层 (External Services)
托管、字体和插件市场等外部依赖
Cloudflare PagesGoogle FontsClaude Code Marketplace
▼
Impeccable 依赖少量外部服务：Cloudflare Pages 负责托管官网和处理下载请求（免费套餐即可）；Google Fonts 提供官网使用的字体文件（完全免费）；Claude Code Marketplace 是 Anthropic 官方的插件市场，让用户可以一键搜索安装 Impeccable。此外，用户也可以通过 npx skills add 命令行工具一键安装，这条命令会自动检测当前使用的 AI 工具并把文件放到正确的位置。
Workflow

从源码到用户手中：构建流水线

Impeccable 的核心工作流是一条自动化流水线，把人类写好的设计知识，变成各种 AI 工具能理解的指令。

1
编写源文件
在 source/ 目录里用带元数据头部的 Markdown 格式编写设计技能和命令，这是唯一需要人工编辑的地方。
→
2
读取 & 解析
构建脚本扫描所有源文件，解析元数据头部（名称、描述、参数等），提取正文内容和引用文件。
→
3
多平台转换
6 个翻译器分别处理：替换占位符、转换格式（Markdown/TOML）、调整参数语法，生成两套版本（标准 + 前缀）。
→
4
打包 & 编译官网
把每个平台的成品打成 ZIP 包，同时编译 Tailwind 样式表，用 Bun 打包官网的静态资源，生成 API 数据文件。
→
5
部署 & 分发
官网部署到 Cloudflare Pages，dist/ 提交到代码仓库。用户通过官网下载 ZIP 或用命令行一键安装。

Glossary

术语词典

页面里出现的技术名词，用大白话解释一遍。

Skill (技能)
AI 编码工具的"专业技能包"。就像给实习生一本《设计入门手册》，让它在特定领域变得更专业。技能文件告诉 AI 该怎么做、不该怎么做。
Frontmatter (元数据头部)
文件最顶部用 --- 包裹的那段信息，像是一封信的"信封" -- 写着收件人、主题等基本信息。AI 工具靠这段信息知道这个技能叫什么名字、干什么用。
Markdown
一种超简单的文本排版格式。用 # 表示标题、** 表示加粗、- 表示列表。程序员写文档最爱用的格式，几分钟就能学会。
Transformer (转换器)
这里不是 AI 模型那个 Transformer。在这个项目里，指的是一段"翻译程序"，负责把通用的源文件转换成特定 AI 工具能理解的格式。6 个翻译器各管一种工具。
Anti-Pattern (反面模式)
"千万别这样做"的设计范例。比如用 Inter 字体、紫色渐变、卡片套卡片 -- 这些都是 AI 生成界面的"指纹"，看一眼就知道是机器做的。
OKLCH
一种现代的颜色表示方式。传统的 RGB/HSL 在调亮度时颜色看起来不均匀，OKLCH 修正了这个问题 -- 同样亮度值的不同颜色，看起来确实一样亮。
Bun
一个"全能运动员"级别的编程工具。别人需要 Node.js + npm + webpack 三件套才能干的活，Bun 一个人就搞定了，而且速度快好几倍。
Cloudflare Pages
一个云端网站托管平台。把你的网站文件传上去，它负责让全世界的人都能快速访问。免费套餐就能满足大多数个人项目的需求。
Tailwind CSS
一套"积木式"的样式工具库。不用自己从零写 CSS 代码，直接在页面元素上贴标签（比如 "text-red" "p-4"）就能快速拼出好看的界面。
CDN
内容分发网络，像快递公司在全国各地设立的分仓。你的文件被复制到世界各地的服务器上，用户访问时从最近的"仓库"取货，所以加载很快。
API
两个软件之间打电话用的"接线员"。比如官网想获取技能列表，就给服务器的 API "打个电话"，API 把数据整理好回传。
YAML
一种让人类容易读写的数据格式，用缩进表示层级关系。比 JSON 更直观，常用来写配置文件。Impeccable 的元数据头部就是 YAML 格式。
TOML
另一种配置文件格式，用方括号和等号来组织数据（比如 [section] key = "value"）。Gemini CLI 用 TOML 格式来定义命令。
Playwright
"AI 的机器人手" -- 一个可以自动打开浏览器、点击按钮、截取屏幕的工具。在这个项目里用来自动截取反面设计案例的截图。
WCAG
网页无障碍标准，像建筑的"无障碍通道规范"。规定了文字对比度至少多少、按钮至少多大等要求，确保色盲、视力不好的人也能正常使用网页。
A11y (无障碍)
Accessibility 的缩写（a 和 y 之间有 11 个字母），指让所有人（包括残障人士）都能使用的设计原则。比如确保文字够大、颜色对比够强。
ZIP
一种把很多文件压缩打包成一个文件的格式，像把一摞文件装进一个信封。下载后解压就能得到里面所有的文件。
npx
一个"临时快递员"命令。不用事先安装某个工具，直接跑 npx 就能临时下载并执行它。npx skills add 就是临时调用安装工具来装技能包。
