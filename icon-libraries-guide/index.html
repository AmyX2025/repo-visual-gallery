为什么是这 7 个？ 不是所有开源图标库都值得用。以下是把 Font Awesome、Material Icons 等大量库排除在外的判断逻辑。
01
没有「认出来了」感
Font Awesome 和 Material Icons 用了超过 10 年，一眼就能认出，会让设计显得平庸。这 7 个库大多数用户不会主动注意到——这才是好图标应有的状态。
02
不是 AI 的默认选项
AI 生成 HTML 时几乎默认使用 Font Awesome 或 Material Icons，因为训练数据里充满了它们。本页收录的都是人工精选后使用的库，天然带有「非 AI 生成」的气质。
03
2018 年之后的美学
图标的流行审美在 2018–2020 年间发生了明显转变，从厚重填充转向精细描边。这 7 个库都诞生或成型于这次转变之后，视觉语言更现代。
04
开发者社区的真实使用
这些库在 GitHub Star、Reddit、开发者博客中持续被提及和推荐，是设计品味好的工程师在实际项目中真正选择的工具，而不是教程里的占位符。
05
设计系统级的一致性
每个库内部都遵守严格的网格规范（24x24 或 15x15），stroke-width、圆角、间距保持统一。混用同一套库的不同图标，不会出现「风格不搭」的问题。
全部免费 / 个人 & 商业均可用
7 个库全部采用 MIT、ISC 或 Apache 2.0 许可证。个人项目、商业产品、SaaS、客户交付物均可免费使用，无需署名，无需付费。
