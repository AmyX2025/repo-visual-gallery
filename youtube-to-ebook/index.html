YouTube 转电子书

自动监听 YouTube 频道更新，把最新视频字幕改写成杂志风格文章，打包成电子书发到你邮箱 — 每周自动跑一次，再也不用追更了

🐍Python 流水线
🤖Claude AI 改写
📧EPUB 邮件推送
⏰macOS / GitHub Actions

架构分层

从用户界面到外部服务，6 层结构让整个系统各司其职

1

用户层

你跟系统打交道的两个入口：一是双击打开仪表盘配置频道和写作风格，二是每周收邮件看电子书
🖥️ 交互入口
2

自动化调度层

com.youtube.newsletter.plist newsletter.yml
macOS 的 launchd 或 GitHub Actions 按时间表定期启动流水线 — Mac 用户本地跑，其他人可以用云端跑（但 YouTube 可能封云服务器 IP）
⏰ 定时任务调度
3

编排器

main.py video_tracker.py
main.py 是总指挥，按顺序调用各个模块；video_tracker.py 负责记住哪些视频处理过了，避免重复发送
🎯 流程控制 + 去重追踪
4

流水线模块

get_videos.py get_transcripts.py write_articles.py send_email.py
四个核心模块接力完成整个过程：抓视频列表 → 扒字幕 → AI 改写 → 打包发邮件。get_videos 会用 HTTP HEAD 请求检查视频 URL 是否重定向到 /shorts/ 来过滤短视频
⚙️ 核心处理流程
5

仪表盘

dashboard.py launch_dashboard.command
Streamlit 做的网页界面，深色杂志风格，5 个页面：一键生成、频道管理、写作风格、历史归档、定时设置 — 双击 launch_dashboard.command 就能打开
🎨 可视化配置界面
6

外部服务

项目依赖四个外部服务：YouTube Data API 查视频列表，Supadata 扒字幕，Claude AI 改写文章，Gmail SMTP 发邮件
🌐 字幕提取服务 + AI 写作助手 + 邮件发送

工作流程

从抓取到发送，5 个步骤全自动完成

📹

抓取视频

去 YouTube 查你订阅的频道最新上传了哪些视频，按时间排序，自动过滤掉短视频

📝

扒字幕

把视频 ID 发给字幕提取服务，拿回完整文字稿 — 每个请求之间等 1 秒避免被限流

✨

AI 改写

把字幕扔给 Claude AI，让它按照你设定的杂志风格重新写一遍，变成可读性强的文章

📚

打包电子书

用 ebooklib 把所有文章打包成 EPUB 格式电子书，顺便生成一个 HTML 邮件预览

📧

发邮件

通过 Gmail SMTP 把电子书作为附件发到你邮箱 — 成功发送后才会标记视频已处理

对话式解读

用聊天的方式讲清楚核心机制

👤
点击"一键生成"按钮后，系统到底做了什么？
🤖
main.py 会按顺序调用 5 个模块：

第一步 — get_videos.py 去 YouTube 查你订阅的频道最新上传了哪些视频，拿到视频 ID、标题、发布时间这些信息 去 YouTube 的视频仓库里查最新上传列表，按时间顺序排好 第二步 — video_tracker.py 检查这些视频 ID 有没有处理过，把新视频筛出来 看看本地的 JSON 记录文件里有没有这些视频的 ID 第三步 — get_transcripts.py 把新视频的 ID 发给字幕提取服务，拿回完整文字稿 每次请求之间停 1 秒，避免请求太快被服务商限流 第四步 — write_articles.py 把字幕发给 Claude AI，让它按照你设定的杂志风格重新写一遍 把字幕和你在仪表盘里设置的写作风格一起发给 AI 模型 第五步 — send_email.py 把所有文章打包成 EPUB 电子书，通过 Gmail 发到你邮箱，成功后才标记视频已处理 用 ebooklib 库生成电子书文件，再用 Gmail 的 SMTP 服务器发送
👤
如果中间某一步失败了怎么办？
🤖
系统只有在最后发送邮件成功之后才会标记视频已处理。如果中间任何一步出错（比如 API 限流、网络超时、AI 服务暂时不可用），video_tracker 不会记录这些视频，下次运行时会重新处理它们。这样可以确保你不会漏掉任何一期内容。
👤
系统怎么知道哪些视频发过了，不会重复发送？
🤖
video_tracker.py 在本地维护了一个 JSON 文件，里面记录了所有成功发送过邮件的视频 ID。每次抓到新视频列表后，会先跟这个文件里的记录对比，只处理那些没见过的 ID。 就像一个签到表，发送成功才打勾，下次看到打过勾的就跳过
👤
如果我想重新处理某个视频呢？
🤖
直接编辑或删除本地的 JSON 追踪文件，把对应的视频 ID 去掉就行。下次运行时系统会把它当作新视频重新处理。不过要小心别误删了其他记录。
👤
YouTube Shorts 短视频也会被抓取吗？它们内容太短不适合做成文章
🤖
系统会自动过滤掉短视频。get_videos.py 用了一个巧妙的方法：对每个视频 URL 发一个轻量级的 HTTP HEAD 请求，检查 YouTube 会不会把它重定向到 /shorts/ 路径。如果重定向了，说明这是个短视频，直接跳过不处理。 就像敲门问一句"你是短视频吗"，如果门牌号变成了 /shorts/ 开头，就知道是了
👤
为什么不直接看视频时长来判断？
🤖
因为 YouTube Data API 返回的视频列表里，短视频和普通视频混在一起，API 响应里没有明确的"这是 Shorts"标记。如果要通过时长判断，需要额外调用一次 API 查询视频详情，会消耗更多配额。用 URL 重定向检查只需要一个 HTTP HEAD 请求，速度快、不消耗 API 配额，而且准确度 100%。

工具箱

支撑整个项目的第三方服务

📺

YouTube Data API v3

你的 YouTube 数据管家 — Google 官方的视频信息查询接口
让你的代码能查频道最新上传了哪些视频、视频标题是什么、什么时候发布的。就像给你开了一个后台查询权限，不用手动刷页面。
免费额度：每天 10,000 配额单位（查询视频列表每次消耗 1 单位，够个人用）
超出后：需要申请提额或付费，但对于监听几个频道来说基本用不完
💬

Supadata API

你的字幕提取专家 — 专门干"把视频变成文字稿"这件事的服务
你给它视频 ID，它给你完整字幕文本。省去了自己写爬虫或者调用 YouTube 字幕接口的麻烦，而且支持多语言、自动处理时间轴。
免费额度：每月 100 次请求
付费套餐：每月 $17 可用 3000 次（约每次 0.006 美元）
算笔账：如果你关注 5 个频道，每周共发 10 个视频，一个月 40 个视频，免费额度够用
✨

Anthropic API (Claude Sonnet 4)

你的私人杂志编辑 — Anthropic 公司做的 AI 写作助手
把口水话满天飞的视频字幕改写成结构清晰、可读性强的杂志文章。你可以设定写作风格（比如"轻松幽默"或"严肃专业"），它会按照你的要求重新组织内容。
输入价格：约每百万字 $4
输出价格：约每百万字 $20
新用户福利：注册送 $5 免费额度
算笔账：一个 10 分钟视频字幕约 2000 字，AI 输出文章约 1500 字，成本不到 $0.01
📧

Gmail SMTP

你的邮递员 — Google 邮箱自带的邮件发送服务
不需要单独买邮件发送服务，直接用你的 Gmail 账号就能发送电子书。需要在 Google 账户设置里生成一个应用专用密码（因为开了两步验证后普通密码不能用于第三方应用）。
免费额度：每天 500 封邮件
限制：附件大小不能超过 25MB（EPUB 电子书一般几百 KB 到几 MB，完全够用）
📚

ebooklib

你的电子书装订机 — 开源的 EPUB 制作工具
一个 Python 库，把文章内容、标题、封面打包成标准的 EPUB 格式电子书。支持自定义样式、章节结构、元数据，生成的文件可以在 Apple Books、Kindle、其他阅读器里打开。
完全免费：开源项目，没有任何费用
安装：pip install ebooklib 就能用
🎨

Streamlit

你的网页搭建助手 — 专门给数据科学家用的快速建站工具
用纯 Python 代码就能做出好看的网页界面，不需要写 HTML/CSS/JavaScript。项目用它做了一个仪表盘，可以管理频道订阅、设置写作风格、查看历史记录。
本地运行：完全免费，开源项目
云端部署：Streamlit Community Cloud 提供免费托管（公开项目）
⚙️

GitHub Actions

你的云端定时任务机器人 — GitHub 自带的自动化执行服务
可以设定每周几点自动运行脚本，不需要自己的电脑一直开着。但是项目作者发现 YouTube 会封禁云服务器 IP，所以现在主要推荐用 macOS 的 launchd 在本地跑。
公开仓库：完全免费，无限分钟
私有仓库：每月 2000 分钟免费额度（够用）
注意：YouTube 可能封云端 IP，建议本地运行
🖥️

macOS launchd

你的 Mac 系统管家 — macOS 自带的定时任务系统
类似 Windows 的任务计划程序或 Linux 的 cron，可以设定每周几点自动运行脚本。项目提供了一个 .plist 配置文件，放到指定目录就能让系统定时启动流水线。
完全免费：macOS 系统自带功能
优势：本地运行，不受云端 IP 封禁影响

术语表

看不懂的词都在这里

API
应用程序接口，简单说就是"让不同软件互相说话的规则"。比如你的代码想问 YouTube"某个频道最新上传了啥"，就通过 YouTube 提供的 API 发请求，它回复你数据。
API 密钥
用来证明"你有权限调用这个 API"的一串字符。就像门禁卡，服务商给你发一个，你每次请求带上它，对方才认你。一般放在 .env 文件里，不能泄露。
SMTP
简单邮件传输协议，发邮件用的标准方法。Gmail、Outlook 等邮箱服务都提供 SMTP 服务器地址，你的代码连上去，按协议格式发送，对方就能收到邮件。
Token
AI 模型处理文本的最小单位，可以粗略理解为"字"或"词块"。中文一般 1 个汉字约等于 1.5-2 个 token，英文一个单词约 1-2 个 token。API 计费按 token 数算。
.env 文件
存放敏感配置信息的文本文件，比如 API 密钥、邮箱密码。文件名以点开头，通常会加到 .gitignore 里防止上传到 GitHub 泄露。程序运行时读取里面的变量。
JSON
一种轻量级的数据格式，用大括号、中括号、冒号、逗号组织数据，人和机器都容易读。项目用 JSON 文件记录处理过的视频 ID。
launchd
macOS 系统的任务调度守护进程，负责在特定时间或条件下启动程序。通过 .plist 配置文件告诉它"每周日晚上 8 点运行这个脚本"。
GitHub Actions
GitHub 提供的自动化服务，可以在云端虚拟机里运行你的代码。常用于自动化测试、部署、定时任务。配置文件放在 .github/workflows/ 目录下。
Streamlit
一个 Python 框架，用纯 Python 代码就能做出交互式网页应用，不需要写前端代码。运行后会在本地启动一个网页服务器，浏览器打开就能用。
流水线 (Pipeline)
按顺序执行的一系列处理步骤，前一步的输出是后一步的输入。这个项目的流水线是：抓视频 → 扒字幕 → AI 改写 → 打包 → 发邮件。
HTTP HEAD 请求
一种轻量级的 HTTP 请求方式，只获取响应头信息，不下载完整内容。用来检查 URL 是否有效、是否重定向、文件大小等，速度快、省流量。
应用专用密码
Google 账户开启两步验证后，第三方应用不能用普通密码登录，需要在账户设置里生成专门给应用用的密码。16 位字符，只显示一次，记得保存到 .env 文件里。
YouTube Data API
Google 提供的官方接口，让开发者能查询视频信息、频道信息、播放列表等。需要在 Google Cloud Console 创建项目、启用 API、获取密钥才能用。
Supadata
一个第三方服务，专门提供视频字幕提取功能。输入视频 URL 或 ID，返回完整文字稿。比自己写爬虫省事，而且处理了多语言、格式转换等细节。
Claude AI
Anthropic 公司开发的大语言模型，擅长理解和生成文本。项目用的是 Claude Sonnet 4 版本，用来把口水话字幕改写成结构清晰的文章。
ebooklib
一个 Python 库，用来读写 EPUB 格式电子书。可以设定章节、样式、元数据、封面等，把文本内容打包成标准的 EPUB 文件。
