Frontend Slides
到底是个什么东西？

一句话版本：它是一个 AI 助手的「技能插件」，能帮完全不会写代码的人，用对话做出漂亮的网页版 PPT。不需要装任何软件，最后生成一个 HTML 文件，双击就能打开。

SKILL.md ← 大脑，指挥一切的总地图
STYLE_PRESETS.md ← 12 套设计风格的「服装店」
viewport-base.css ← 保证不会出滚动条的安全网
html-template.md ← HTML 骨架的图纸
animation-patterns.md ← 动画特效的菜谱
scripts/extract-pptx.py ← PowerPoint 的翻译官
技术架构

这个项目的「楼层图」

从上到下，每一层干不同的活。点击任意一层看详细说明。

1
🧑 你（用户）
打开终端，输入 /frontend-slides，告诉 AI 你想做什么样的 PPT。
你不需要会写代码，不需要懂设计。你只需要有想法 -- 比如「帮我做一个创业融资演讲」或者「把这个 PPT 文件转成网页版」。AI 会像一个耐心的设计师一样，一步步问你问题，给你看样式预览，最后帮你生成成品。
↓
2
🤖 Claude Code（AI 引擎）
Anthropic 公司出品的 AI 编程助手，负责读懂技能文件、执行指令、生成代码。
claude.ai/claude-code
把 Claude Code 想象成一个超级实习生 -- 它能读懂文档、写 HTML/CSS/JavaScript 代码、运行 Python 脚本、打开浏览器预览文件。它本身不知道怎么做幻灯片，但读了 SKILL.md 这份「操作手册」之后，就变成了一个幻灯片专家。这就像一个聪明的新员工，看完公司手册就能上岗。
↓
3
🧠 SKILL.md（大脑 / 总地图）
只有约 180 行的精炼指令，像一张地图一样告诉 AI：先做什么、再做什么、什么时候该去读哪个文件。
SKILL.md
这是整个项目最核心的设计理念：给地图，不给百科全书。SKILL.md 不会一口气把所有信息塞给 AI，而是说「你现在需要选风格？去读 STYLE_PRESETS.md」「要生成代码？去读 html-template.md 和 viewport-base.css」。这叫做渐进式加载 -- 用到什么才读什么，就像你点外卖不会一次把菜单全背下来。
↓
4
📚 辅助参考文件（按需加载）
风格预设、CSS 安全网、HTML 模板、动画菜谱 -- AI 需要哪个就读哪个，而不是一次全塞进去。
STYLE_PRESETS.md viewport-base.css html-template.md animation-patterns.md
STYLE_PRESETS.md -- 12 套精心设计的视觉风格，从「霓虹赛博」到「手账本风」都有，每套包含字体搭配、配色方案和标志性元素。

viewport-base.css -- 一段必须被复制到每个演示文稿里的CSS代码。它的作用是确保每一页幻灯片刚好填满屏幕，不会出现烦人的滚动条。

html-template.md -- 生成 HTML 文件的「图纸」，规定了文件结构、键盘导航、触摸滑动、进度条等功能怎么实现。

animation-patterns.md -- 各种动画效果的代码片段，并且标注了每种动画适合什么「情绪」。
↓
5
🎨 输出成品（一个 HTML 文件）
所有的代码、样式、动画、字体都打包在一个文件里。双击就能在浏览器打开，十年后照样能用。
presentation.html
最终产出是一个零依赖的 HTML 文件 -- 不需要安装任何软件，不需要联网（除了加载字体），不需要 npm、webpack 或者任何构建工具。项目的哲学是「依赖就是债务」：一个单独的 HTML 文件十年后还能打开，但一个 2019 年的 React 项目？祝你好运。这个文件里包含了键盘导航（方向键翻页）、触摸滑动（手机上手指左右滑）、进度条、导航圆点等全部功能。
工作流程

从「我要做 PPT」到「成品出炉」

整个过程分 5 个阶段，就像去定制西装一样：先量体、选面料、再裁剪。

0

判断模式

你是想从零开始做？还是把现有 PPT 转成网页版？还是想改进已有的？

1

了解需求

AI 问你：做什么用？多少页？内容准备好了吗？要不要在线编辑功能？

2

选风格

生成 3 个单页预览。你说不清楚喜欢什么？没关系，看到再选就行。

3

生成代码

AI 读取模板和样式，一口气生成完整的 HTML 演示文稿。

4

交付成品

清理临时文件，自动打开浏览器预览，告诉你怎么自定义。

「看到再选」的设计哲学

为什么不直接让你描述想要的风格？

手册
SKILL.md 的设计哲学
大多数人说不清楚自己喜欢什么风格。你让他们选「现代」还是「经典」？他们会一脸茫然。但如果你给他们看三个成品，他们会立刻指着其中一个说「就这个！」
AI
Claude Code
明白了。所以我不问「你喜欢什么风格」，而是问「你想让观众有什么感觉」？
风格
STYLE_PRESETS.md
没错！我这里有 12 套预设风格，每套都有一个「情绪标签」。比如想让人觉得「被震撼到了」？用 Bold Signal 或 Neon Cyber。想让人「安心、信任」？用 Paper & Ink 或 Swiss Modern。我不只是一个色板表，我是情绪到视觉的翻译器。
CSS
viewport-base.css
而且不管选哪个风格，我都会被完整复制到最终文件里。我的工作是确保每一页幻灯片刚好填满屏幕，在任何设备上都不会出现滚动条。我是所有风格共用的「安全网」。
动画
animation-patterns.md
我负责给每种情绪配上合适的动画。想要「戏剧感」？用慢速淡入 + 大比例缩放。想要「科技感」？用霓虹发光 + 文字乱码特效。想要「友好亲切」？用弹性动画 + 浮动效果。不是为了炫技，是为了强化你想传达的感觉。
手册
SKILL.md
还有一条铁律：绝对不做「AI 罐头味」的东西。不用 Inter 字体、不用紫色渐变白底、不用千篇一律的卡片网格。每个演示文稿都要像专门为你设计的，不是从模板库里拽出来的。

术语词典

那些「听着很唬人」的技术词

每个术语用大白话解释一遍，不装、不绕。

HTML
网页的「骨架」。就像盖房子时的砖头和框架，决定了页面上有什么内容（标题、段落、图片），但不管好不好看。你双击打开的那个 .html 文件，就是这个东西。
CSS
网页的「化妆师」。HTML 负责内容，CSS 负责好看 -- 颜色、字体、间距、动画效果全靠它。项目里的 viewport-base.css 就是一段 CSS 代码。
JavaScript (JS)
网页的「行为能力」。键盘按方向键能翻页、手指滑动能切换、进度条会跟着走 -- 这些互动功能都是 JavaScript 实现的。
viewport (视口)
你的屏幕能看到的那个「窗口区域」。项目里反复强调「每页幻灯片必须刚好填满 viewport」，意思就是不能出现滚动条，一屏就是一页。
clamp()
一个 CSS 函数，像「智能弹簧」。你给它一个最小值、一个理想值、一个最大值，它会根据屏幕大小自动调节。比如字号在手机上小一点、在大屏上大一点，不用你操心。
Intersection Observer
浏览器内置的一个「哨兵」。它蹲在那里看：这个元素进入屏幕了吗？进了就触发动画。比翻页时整页「淡入」效果就是靠它实现的，而且比老方法省电。
scroll-snap
「吸附式滚动」。普通网页你滚到哪是哪，但幻灯片需要翻一页就精确停在下一页的开头。scroll-snap 就是让页面自动「咔哒」一声对齐的功能。
零依赖 (Zero Dependencies)
不需要安装任何额外软件或框架就能运行。就像一份纸质文件，不需要专门的阅读器。这个项目生成的 HTML 文件就是零依赖的 -- 双击就打开，十年后依然能用。
CDN
Content Delivery Network，内容分发网络。就像快递公司在全国各地都有仓库，Google Fonts 把字体文件存在世界各地的服务器上，你的浏览器会自动从最近的仓库取货，加载更快。
MIT 协议 (MIT License)
一种超级宽松的开源协议。简单说就是「随便用，商用也行，改了也行，唯一要求是保留版权声明」。这个项目和它用到的 python-pptx 库都是 MIT 协议。
渐进式加载 (Progressive Loading)
需要什么才加载什么，不一口气全塞进来。就像去图书馆查资料，你不会把整个图书馆搬回家，而是需要哪本书才借哪本。这个项目的 AI 只在需要时才读取辅助文件。
prefers-reduced-motion
一个「礼貌的询问」。有些人对闪烁的动画敏感（比如有前庭障碍的用户），他们可以在系统设置里关掉动画。这段代码就是说：「如果用户不想要动画，我就把动画关掉」。
