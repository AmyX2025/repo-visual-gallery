frontend-slides

一个 Claude Code 技能插件，让 AI 通过对话就能生成零依赖的幻灯片演示文件 — 单个文件包含样式、动画和交互逻辑，十年后也能打开
📁 6 个核心文件
🎨 12 种风格预设
⚡ 零依赖输出
🔗 GitHub 仓库

系统架构

从用户输入到浏览器展示，一共经过五层处理流程
1
用户层
在终端输入 /frontend-slides 命令，就像点了个开关
用户只需要描述想法，比如"给团队做个技术分享 PPT"或者"把这个 PowerPoint 转成网页版"
2
AI 引擎
Claude Code（AI 助手）开始工作，像个项目经理一样读取操作手册
它会自动判断你想做什么，然后逐步引导对话 — 有点像填表格，但比填表格自然多了
3
操作手册与资源库
SKILL.md 是大脑（告诉 AI 该做什么），配套文件是工具箱
SKILL.md — 180 行工作流指令，定义了 AI 的每一步行为
STYLE_PRESETS.md — 12 套风格预设，像服装店的试穿间
viewport-base.css — 必备基础样式，确保每张幻灯片正好填满一屏
html-template.md — 文件结构蓝图，定义键盘导航、触摸滑动等功能
animation-patterns.md — 动画图鉴，根据情绪匹配动画效果
extract-pptx.py — PowerPoint 解析脚本，能读取 .pptx 文件内容
4
代码生成
AI 根据你的选择，组装 HTML、CSS、JavaScript 代码
所有代码写进一个文件，字体从 Google Fonts 或 Fontshare 加载，除此之外零外部依赖 — 不需要 npm install，不需要 node_modules 文件夹
5
浏览器展示
生成的 HTML 文件可以直接用浏览器打开，支持键盘翻页、触摸滑动、进度条显示
文件可以永久保存，十年后依然能打开 — 因为它只用浏览器原生功能，不依赖任何会过时的第三方库

工作流程

AI 按照五个阶段逐步完成任务，每个阶段都有明确目标
Phase 0
判断模式

检测用户想做什么：从零创建、转换已有的 PowerPoint、还是修改现有演示文稿

Phase 1
了解需求

询问演示目的、预计长度、内容是否准备好、是否需要可编辑的幻灯片备注

Phase 2
选风格

生成 3 个单页预览供用户挑选 — 不问"你想要什么风格"，而是直接展示选项让你选

Phase 3
生成代码

读取 HTML 模板、基础样式和动画库，组装成完整的演示文稿代码

Phase 4
交付成品

保存文件、自动打开浏览器预览，并建议可能的后续定制选项

AI
为什么不直接问"你想要什么风格"？

因为大多数人说不清楚自己想要什么风格。你可能会说"简洁一点"、"专业一点"、"酷一点"，但这些词太模糊了 — 我生成的东西可能跟你脑子里想的完全不一样。

所以项目采用"先给选项，再让你挑"的方式。我会根据你的场景（技术分享、产品介绍、学术报告等），从 STYLE_PRESETS.md 的 12 个预设里挑 3 个最合适的，直接生成预览页面给你看。
（SKILL.md 的核心哲学："Show don't tell" — 展示选项比询问偏好更有效）
AI
STYLE_PRESETS.md 里有什么？

12 套精心设计的风格预设，每个包含：
• 字体搭配（从 Google Fonts 和 Fontshare 挑选）
• 配色方案（主色、辅助色、背景色）
• 情绪标签（比如"冷静专业"、"活力创新"、"优雅复古"）
• 适用场景建议

比如"Neon Cyber（霓虹赛博）"适合科技产品发布，"Paper & Ink（纸墨）"适合学术报告，"Dark Botanical（暗黑植物）"适合艺术设计展示。
（STYLE_PRESETS.md 用"情绪 + 视觉元素"的方式描述风格，AI 能快速匹配用户场景）
AI
viewport-base.css 的作用

这是个"安全网"。它确保无论你的内容有多长、浏览器窗口多大，每张幻灯片都正好占满一屏，不会出现滚动条或者内容溢出。

里面用了 viewport 单位（vh 和 vw）、scroll-snap 属性（让翻页有吸附效果）、clamp() 函数（让字体大小在不同屏幕上都合适）。所有生成的幻灯片都必须包含这个基础样式。
（viewport-base.css 是强制规则，防止生成的幻灯片在不同设备上显示异常）
AI
animation-patterns.md 如何工作

它是个"情绪 → 动画"的映射表。比如：
• 严肃正式的场景 → 用淡入效果，避免花哨动画
• 活力创新的场景 → 用弹性动画、滑动进入
• 优雅艺术的场景 → 用缓慢渐现、视差滚动

AI 会根据你选的风格预设，自动匹配对应的动画模式。还会考虑 prefers-reduced-motion 设置，尊重用户的动画偏好。
（animation-patterns.md 确保动画效果与内容氛围一致，不是为了炫技而加动画）

术语表

项目里出现的技术名词解释，点击正文中的术语可快速查看定义
HTML
超文本标记语言，网页的骨架。用标签（比如 <h1>、<p>、<div>）来描述内容结构，浏览器读懂后就知道该显示什么。
CSS
层叠样式表，网页的化妆师。决定元素的颜色、字体、布局、动画效果。viewport-base.css 就是一个 CSS 文件。
JavaScript
网页的行为控制器，让页面能响应用户操作。比如按键盘左右箭头翻页、点击按钮触发动画，都是 JavaScript 实现的。
viewport
视口，就是浏览器窗口里能看到内容的那块区域。viewport-base.css 确保每张幻灯片正好占满一个视口，不多不少。
clamp()
CSS 函数，用来限制数值范围。比如 clamp(16px, 2vw, 24px) 表示"字体大小随屏幕宽度变化，但最小 16px，最大 24px"，让文字在不同设备上都合适。
Intersection Observer
浏览器提供的 API，用来检测元素是否进入视口。常用于"滚动到某个位置才触发动画"的效果，比监听滚动事件更高效。
scroll-snap
CSS 属性，让滚动有"吸附"效果。比如翻页时，页面会自动吸附到最近的一张完整幻灯片，不会停在中间位置。
零依赖
指生成的 HTML 文件不依赖任何第三方 JavaScript 库（比如 React、Vue、jQuery），只用浏览器原生功能。好处是文件小、加载快、永远不会因为库过时而打不开。
CDN
内容分发网络，把文件部署到全球各地的服务器上，用户访问时自动选最近的服务器下载，速度更快。Google Fonts 和 Fontshare 都用 CDN 分发字体文件。
MIT 协议
一种非常宽松的开源许可证，允许你自由使用、修改、商用代码，唯一要求是保留原作者的版权声明。python-pptx 就用这个协议。
渐进式加载
SKILL.md 的核心策略 — 不是一开始就把所有文件都读进来，而是根据对话进度，只在需要时才加载对应的文件。就像"给地图，不给百科全书"。
prefers-reduced-motion
浏览器提供的用户偏好设置，有些人会因为动画感到不适（比如眩晕），他们可以在系统设置里打开"减少动画"。网页可以通过 CSS 检测这个设置，自动关闭或简化动画。
Claude Code
Anthropic 公司出品的 AI 编程助手，能在终端里运行，理解自然语言、读写代码、执行命令。frontend-slides 是它的一个技能插件。
Skill（技能插件）
Claude Code 的扩展机制，就像给 AI 装上专用工具。一个 Skill 包含指令文件（SKILL.md）和配套资源，告诉 AI 如何完成特定任务。
